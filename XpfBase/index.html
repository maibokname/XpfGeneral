XpfBase: eXPerimental Fortran Base Package

XpfBase is an experimenal package written in Fortran. It is a sub-package of the XpfGeneral package (a larger package within the XpfLib library), which contains a large collection of general-purpose modules. The XpfBase package consists of various basic and utility modules that are commonly used in other sub-packages of the XpfGeneral package.

The source code of the XpfBase package is organized into 9 subfolders of the XpfBase folder. A brief overview of the source code in each subfolder is given below.

  1. The Abstract subfolder contains two modules that provide two abstract derived types (ADT) that can be employed as base types for other derived types with object-oriented programming (OOP) implementation.

  2. The BinDecConv subfolder contains three modules that provide various binary-to-decimal and decimal-to-binary conversion routines for 32-bit, 64-bit and 128-bit floating point (real) numbers.

  3. The Classes subfolder contains several modules that provide useful derived types implemented with object-oriented programming.

  4. The Containers subfolder contains two subfolders. The Intrusive subfolder contains several modules/submodules that provide various intrusive-based container types. The StackList subfolder contains several modules that provide stack container types for common intrinsic data types.

  5. The Core subfolder contains several core modules that are frequently used by other modules.

  6. The Int128Bits subfolder contains two modules (and their many submodules) that provide derived types and their related routines for 128-bit signed and unsigned integers.

  7. The MemPool subfolder contains several core modules for memory handling/management.

  8. The PolyForm subfolder contains two basic modules for dealing with unlimited polymorphic entities.

  9. The Util subfolder contains a number of modules that provide auxiliary functionalities for other modules.

Dependencies

By default, the XpfBase pacakge is configured to be dependent on the Xpf3rdPT package, which contains a collections of third-party libraries. However, the XpfBase pacakge can be configured to be an independent package by commenting out macro definitions that define the uses of the third-party libraries. In particular, the modules that may or may not be dependent on third-party libraries are described below.

  • The MBase_CppStdUtil module totally depends on the FLibCpp library, which provides Fortran bindings to the C++ Standard Library. By commenting out the macro definition specified at the beginning of the file, the entire module will not be compiled, and it, thus, will not be available.

  • The MBase_LegacyUtil module partially depends on the DispModule library, which provides pretty printing functionalies. By commenting out the macro definition specified at the beginning of the file, some routines depending on the library will not be available.

  • The MBase_MiscUtil module partially depends on the FMLib and MPFun libraries, which provide a variety of multi-precision computations. By commenting out the macro definition specified at the beginning of the file, some routines depending on these libraries will not be available.
